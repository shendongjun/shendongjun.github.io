7.3、乐观锁

主要适用场景：当要更新一条记录的时候，希望这条记录没有被别人更新，也就是说实现线程安全的数 据更新
乐观锁实现方式：
取出记录时，获取当前version
更新时，带上这个version
执行更新时， set version = newVersion where version = oldVersion
如果version不对，就更新失败
（1）数据库中添加version字段
（2）实体类添加version字段

（3）元对象处理器接口添加version的insert默认值

值，两个只能取一个

11、Wrapper


Wrapper ： 条件构造抽象类，最顶端父类
AbstractWrapper ： 用于查询条件封装，生成 sql 的 where 条件
QueryWrapper ： Entity 对象封装操作类，不是用lambda语法
UpdateWrapper ： Update 条件封装，用于Entity对象更新操作
AbstractLambdaWrapper ： Lambda 语法使用 Wrapper统一处理解析 lambda 获取 column。 LambdaQueryWrapper ：看名称也能明白就是用于Lambda语法使用的查询Wrapper LambdaUpdateWrapper ： Lambda 更新封装Wrapper

11.1、ge、gt、le、lt、isNull、isNotNull

（1）ge 代表 >=（大于等于）
（2）gt 代表 > （大于）
（3）le 代表 <= (小于等于）
（4）lt 代表 < (小于）
（5）isNull 代表 该值为空
（6）isNotNull 代表 该值不为空
例子：

SQL：UPDATE user SET deleted=1 WHERE deleted=0 AND name IS NULL AND age >= ? AND email IS NOT NULL

11.2、eq、ne

(1)eq 判断是否相等
(2)ne 判断不相等
注意：seletOne返回的是一条实体记录，当出现多条时会报错
(1.1)

SQL:SELECT id,name,age,email,create_time,update_time,deleted,version FROM user WHERE deleted=0 AND name = ?
(2.1)

Execute SQL：SELECT id,name,age,email,create_time,update_time,version,deleted FROM user WHERE deleted=0 AND (name <> ‘Tom’)

11.3、between、notBetween

包含大小边界

Execute SQL：SELECT COUNT(1) FROM user WHERE deleted=0 AND age BETWEEN ? AND ?

11.4、allEq

全等

SELECT id,name,age,email,create_time,update_time,deleted,version FROM user WHERE deleted=0 AND name = ? AND id = ? AND age = ?

11.5、like、notLike、likeLeft、likeRight

(1)like LIKE %like%
(2)notLike NOT LIKE ‘%e%’
(3)likeLeft LIKE ‘%t’
(4)likeRight LIKE ‘t%’

Execute SQL：SELECT id,name,age,email,create_time,update_time,version,deleted FROM user WHERE deleted=0 AND (name NOT LIKE ‘%e%’ AND email LIKE ‘t%’)

11.6 in

in(R column, Collection value)in(boolean condition, R column, Collection value)

  • 字段 IN (value.get(0), value.get(1), …)
  • 例: in(“age”,{1,2,3})—>age in (1,2,3)

in(R column, Object… values)in(boolean condition, R column, Object… values)

  • 字段 IN (v0, v1, …)
  • 例: in(“age”, 1, 2, 3)—>age in (1,2,3)
